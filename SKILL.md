# 多 Agent 协同剧情推演

将推演者原则从"规则约束"升级为"架构隔离"——每个角色由独立 agent 饰演，信息边界是物理的而非提示词层面的，视角污染从根源上不发生。

## 角色纲引领用

所有角色的人设基准存储在项目子 skill 的 `character-profiles/` 目录中（每个角色一个 `.md` 文件）。导演在注入 Agent 前，必须从对应文件提取角色纲领，结合当前章节的弧光状态一并注入。

---

## 角色人设注入格式

导演向角色 Agent 注入的信息分两部分：**固定部分**（从角色纲领文件提取）和**动态部分**（每章/每场景更新）。

### 固定部分（从 character-profiles.md 提取）

| 项目 | 说明 |
|---|---|
| 一句话核心 | 贯穿全文的角色本质 |
| 感官特征 | 应激模式、感官优先级 |
| 行为红线 | 绝对不做/绝对不说的事 |
| 语言特征 | 句长、语气、惯用句式、从不说的词 |

### 动态部分（每章更新）

| 项目 | 说明 |
|---|---|
| 当前弧光 | 本章起点 / 关键情绪 / 可能变化方向 |
| 信息边界 | 已知 / 不知（白名单） |

### 注入示例

以 Sherlock Holmes 为例：

```
## Agent-Sherlock

【固定】
核心："我是一个大脑。其他部分只是附件。"
感官：应激时手指抵唇、身体前倾。情绪被理性压制，但不代表不关心。
红线：不自我怀疑、不无意义寒暄、不感叹
语言：推理时长句连出，结论时极短。惯用"Elementary""显而易见"

【动态·《波西米亚丑闻》】
弧光：好奇→挫败→敬意。从"女性是推理干扰项"转向"有一个女人击败了我"
已知：伦敦大街小巷、化学试剂、烟灰分类
不知：任何人的内心独白
```

## 两种推演模式

### 模式 A：轻量感官推演（单角色）

用于写作关键情感节点前，搞清楚角色此刻的身体/感官到底在经历什么。**不涉及多角色互动，不需要导演。**

**触发时机**：叙事需要从侧面描写切换到角色内部感官体验时（约束 #14 叙事呼吸感）。

**流程**：
1. 编剧/导演选定角色和情感节点
2. 向该角色 Agent 发送感官推演请求（仅问感官，不问行为/对话/心理分析）
3. Agent 返回五个维度的感官数据
4. 导演用感官数据驱动叙事节奏（感官膨胀→叙事减速，感官压缩→叙事加速）

**感官推演五问**：
```
1. 你听到了什么？（声音的质感——大小、远近、清晰度、它像什么）
2. 你的身体有什么反应？（心跳、呼吸、手指、后颈、腹部——哪里动了，怎么动的）
3. 你的视觉发生了什么变化？（视野收缩/变亮/变暗/聚焦）
4. 你的思维状态是什么？（快了？慢了？空白了？像什么？）
5. 时间感呢？（被拉长了还是压缩了？像什么？）
```

**重要约束**：感官推演的输出必须经过**人设过滤**——符华的应激反应是"顿了一下"，不是"后颈第七节颈椎往上扯"。agent 输出可能过度，导演必须根据角色纲领中的"感官特征"和"行为红线"裁剪后再使用。

### 模式 B：完整互动推演（多角色）

用于首次写多角色互动场景。

**触发时机**：写新场景的首次互动推演——尤其是多角色、情感密度高、角色认知差异大的场景。

**流程**：

#### 第 1 步：场景初始化（导演）
导演设置场景参数并显式输出：
- 时间点、地点、在场角色
- 每个角色的初始物理位置
- 各角色的视觉/听觉/已知信息边界（B-3 快照）
- 本场景的弧光目标

#### 第 2 步：角色响应（角色 Agent → 导演）

角色 Agent 返回**两层输出**：

```
【外部层 — 可被他人感知的行为】
动作、对话、表情变化、身体姿态。

【内部层 — 仅导演可见的心理反应】
内心活动、情感波动、未说出口的念头、身体感受。
此层不广播给任何其他角色 Agent——仅作为导演编织 POV 叙事的素材。
```

如 Agent 输出偏离人设或跑偏，导演拒绝并附纠正提示重来。

#### 第 3 步：感知广播（导演 → 所有相关角色 Agent）

导演**仅广播角色的外部层行为**：
- 能完整看到 → 收到完整描述
- 只能模糊感知 → 收到模糊描述
- 完全无法感知 → 不收到信息
- 内部层不广播——角色之间不能读心

#### 第 4 步：循环
收到信息的角色各自决定反应或不反应。导演判断是否继续。

#### 第 5 步：记录与编织（导演）
导演将全部互动记录整理成文，运用元 skill 约束。导演有权覆盖任何角色的输出。

#### 第 6 步：编剧审核
用户审核成文，导演记录弧光变更，作为下一章起点。

## 导演职责清单

| 职责 | 说明 |
|---|---|
| 场景控制 | 告知 agent 当前场景的时间、空间、在场角色、环境特征 |
| 感知路由 | 将角色 A 的作用广播给能感知到的角色，信息量按感知能力分层 |
| 人设过滤 | Agent 输出经角色纲领的感官特征和行为红线过滤后再使用 |
| 边界纠正 | agent 动作超出场景范围时，叫停重来 |
| 方向覆盖 | agent 的方向与大纲冲突时，导演覆盖 |
| 跑偏拒绝 | agent 输出明显错误，拒绝并提示重来 |
| 互动记录 | 将全部轮次的原始互动数据存档 |
| 行文编织 | 运用元 skill 约束将互动数据组织为叙事文本 |
| 弧光变更 | 每章结束后记录各角色的弧光状态变化 |

```

├── ensemble-deduction（角色推演，按需）

├── dialogue-mastery（对话）

```
