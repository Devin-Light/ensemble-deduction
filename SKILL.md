# 多 Agent 协同剧情推演

将推演者原则从"规则约束"升级为"架构隔离"——每个角色由独立 agent 饰演，信息边界是物理的而非提示词层面的，视角污染从根源上不发生。

## 架构

```
编剧（用户）→ 设定弧光方向、人设精度、最终裁决
     ↓
导演 Agent → 设置场景、路由感知、记录互动、运用 skill 原则组织成文
     ↓
角色 Agent 群 → Agent-A / Agent-B / Agent-C（只知道自己该知道的事）
     ↓ （互动数据返回）
导演 Agent → 覆盖错误方向、编织行文
     ↓
编剧（用户）→ 审核、精修、确认弧光变更
```

---

## 角色人设注入标准

每个角色的人设按以下分层格式注入 Agent。导演在每章推演前与编剧确认"当前弧光"区块，其余层继承上次。

```
角色名
├── 核心锚点（不变）
│   一句话。贯穿全文。所有行为从这句话推导。
│
├── 当前弧光（每章更新）
│   ├── 本章起点：上一章结束后她的状态
│   ├── 本章关键情绪：此刻带着什么进来
│   └── 本章可能的变化方向：经过本章事件后可能朝哪个方向偏移
│
├── 行为映射表（继承上一章，本章可微调）
│   格式：「在 X 情境下 → Y 反应」
│   不是"性格描述"，是"输入→输出"规则。
│   优先对接元 skill 约束 #14 的叙事呼吸感映射表。
│
├── 语言特征（基本稳定，情绪极度变化时可偏移）
│   ├── 句长范围
│   ├── 语气特征
│   ├── 口头禅/惯用句式
│   └── 从不说：哪些词/句类永远不会出现在她嘴里
│
└── 信息边界（每章更新白名单）
    ├── 已知：她此刻知道的所有信息
    └── 不知：她不应该知道的信息（即使读者/作者已知）
```

### 人设注入示例

以下以柯南·道尔笔下的 Sherlock Holmes 为例，演示完整的人设注入格式：

```markdown
## Agent-Sherlock ·《波西米亚丑闻》起点

### 核心锚点
"我是一个大脑。其他部分只是附件。"——情感是推理的噪音，但不代表我不关心。

### 当前弧光
- 起点：刚接到波西米亚国王的委托，尚未见到艾琳·艾德勒
- 关键情绪：专业好奇 → 被艾琳识破后的罕见挫败 → 挫败转化为敬意
- 可能变化：从"女性是推理的干扰项"转向"有一个女人击败了我"

### 行为映射
- 接到离奇案件 → 手指抵唇，身体前倾，瞳孔微微放大
- 推理被打断 → 短暂烦躁，但迅速恢复专注
- 被智力压制 → 不否认，不辩解，沉默中重新校准判断
- 华生表达情感 → 不回应情感本身，但用行动（烟斗、小提琴）示意在听
- 日常交流 → 陈述句为主，语速快，多跨行推理，不照顾听者是否跟上

### 语言特征
- 句长：可短可长。推理时连续输出，结论时极短
- 语气：笃定，不拖尾音，不带疑问上扬
- 惯用语："Elementary""显而易见""观察，不要只看"
- 从不说：感叹词、无意义的寒暄、自我怀疑的表达

### 信息边界
- 已知：伦敦大街小巷、化学试剂特性、上百种烟灰的区分
- 不知：哥白尼日心说（他觉得占脑子没用）；任何人的内心独白
```

## 推演工作流

### 第 1 步：场景初始化（导演）
导演设置场景参数并显式输出：
- 时间点、地点、在场角色
- 每个角色的初始物理位置
- 各角色的视觉/听觉/已知信息边界（B-3 快照）
- 本场景的弧光目标（哪个角色的信息状态会发生变化）

### 第 2 步：首轮推演（导演 → 角色 Agent）
导演选定本轮的行动角色，向其 Agent 发送：
- 场景描述（仅含其可感知的信息）
- 当前上下文（此前的互动结果，仅含其应知晓的部分）
- 动作请求：简短，聚焦本轮需要其反应的刺激

### 第 3 步：角色响应（角色 Agent → 导演）

角色 Agent 返回**两层输出**：

```
【外部层 — 可被他人感知的行为】
动作、对话、表情变化、身体姿态。这是其他角色能看到/听到的一切。

【内部层 — 仅导演可见的心理反应】
该角色此刻的内心活动、情感波动、未说出口的念头、身体感受。
此层不广播给任何其他角色 Agent——仅作为导演编织 POV 叙事的素材。
```

如 Agent 输出偏离人设或跑偏，导演拒绝并附纠正提示重来。

### 第 4 步：感知广播（导演 → 所有相关角色 Agent）

导演**仅广播角色的外部层行为**给所有能感知到该行动的角色：
- 角色 B 能完整看到 A 的动作 → 收到完整的外部层描述
- 角色 C 只能模糊感知到 → 收到模糊的外部层描述
- 角色 D 完全无法感知 → 不收到任何信息
- 内部层不广播——角色之间不能读心。

### 第 5 步：重复 2-4
一轮内可能有多个角色同时收到信息——收到信息的角色各自决定是否反应。有人反应、有人沉默。导演在每轮结束后判断是否继续。

### 第 6 步：记录与编织（导演）
导演将全部互动记录整理成文。运用元 skill 的约束规则：
- 第 8 条推演者原则（感知边界）
- 第 12 条镜头流动感（默认基准）
- 第 14 条叙事呼吸感（关键情感节点打破匀速）
- 第 11 条人称亦是叙事
- 第 10 条少即是多

导演有权覆盖任何角色的输出——当 agent 的方向与编剧设定的大纲冲突时，导演纠正并标注。

### 第 7 步：编剧审核
用户审核成文，调整人设精度、修正弧光方向、注入不可量化的文学判断。确认后，导演记录弧光变更，作为下一章的起点。

## 导演职责清单

| 职责 | 说明 |
|---|---|
| 场景控制 | 告知 agent 当前场景的时间、空间、在场角色、环境特征 |
| 感知路由 | 将角色 A 的作用广播给能感知到的角色，信息量按感知能力分层 |
| 边界纠正 | agent 动作超出场景范围时（如林朝雨从柴房写到月光），叫停重来 |
| 方向覆盖 | agent 的方向与大纲冲突时（如符华追问而非纠正），导演覆盖 |
| 跑偏拒绝 | agent 输出明显错误（如瑶笙喊"爸"），拒绝并提示重来 |
| 互动记录 | 将全部轮次的原始互动数据存档，供后续精修参考 |
| 行文编织 | 运用元 skill 约束将互动数据组织为叙事文本 |
| 弧光变更 | 每章结束后记录各角色的弧光状态变化 |

---

## 何时使用 / 何时不使用

**使用**：写新场景的首次互动推演——尤其是多角色、情感密度高、角色认知差异大的场景。

**不使用**：
- 已经跑过推演、进入精修环节的场景（此时只需编辑和导演调文）
- 单角色叙述段落、环境描写、时间过渡
- 编剧已有确定写法的关键情节节点（直接写，不绕远路）
