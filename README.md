# ensemble-deduction

> **版本**：0.1-preview
>
> 多 Agent 协同剧情推演——将推演者原则从"规则约束"升级为"架构隔离"

## 这是什么？

`ensemble-deduction` 是一个 Claude Code skill，用于严肃文学创作中的多角色互动场景推演。它为每个角色分配独立的 Agent，各自注入专属人设和信息边界，由导演 Agent 统一设置场景、路由感知输入、记录互动并组织成文。

**核心创新**：传统写作中，作者需要在脑中同时维护所有角色的信息状态——谁看到什么、谁不知道什么。推演者原则要求"每个角色只能基于自己能感知到的信息做出反应"，但单个 LLM 天然"知道一切"。`ensemble-deduction` 用架构隔离替代规则约束——角色 Agent 的上下文中根本没有其他角色的心理活动，视角污染从根源上不发生。

## 架构

```
编剧（用户）→ 设定弧光方向、人设精度、最终裁决
     ↓
导演 Agent → 设置场景、路由感知、记录互动、运用 skill 原则组织成文
     ↓
角色 Agent 群 → Agent-A / Agent-B / Agent-C（各自信息隔离）
     ↓ （互动数据返回）
导演 Agent → 覆盖错误方向、编织行文
     ↓
编剧（用户）→ 审核、精修、确认弧光变更
```

## 角色人设注入标准

每个角色按五层格式注入 Agent：

```
角色名
├── 核心锚点（不变）
│   一句话。贯穿全文。所有行为从这句话推导。
├── 当前弧光（每章更新）
│   起点 / 关键情绪 / 可能变化方向
├── 行为映射表
│   格式：「在 X 情境下 → Y 反应」
├── 语言特征
│   句长范围 / 语气 / 口头禅 / 从不说
└── 信息边界（每章更新白名单）
    已知 / 不知
```

## 推演工作流

1. **场景初始化**（导演设置场景参数 + B-3 感知快照）
2. **首轮推演**（导演向角色 Agent 发送仅含其可感知信息的场景描述）
3. **角色响应**（Agent 返回动作/对话/沉默；跑偏则导演拒绝并重来）
4. **感知广播**（将角色 A 的行动分层广播给所有能感知到该行动的角色）
5. **循环**（收到信息的角色各自决定反应或不反应）
6. **记录与编织**（导演运用元 skill 约束将互动数据组织成文）
7. **编剧审核**（用户精修，导演记录弧光变更）

## 何时使用 / 何时不使用

**使用**：首次写多角色、情感密度高、认知差异大的互动场景。

**不使用**：精修环节、单角色叙述段落、编剧已有确定写法的关键节点。

## 依赖

本 skill 设计为配合 `literary-fanfic-orchestrator` 元 skill 使用——作为阶段 5（场景写作）中 `scene-craft` 的前置步骤。可独立使用，但推演编织效果依赖于元 skill 中的约束体系。

## 快速开始

1. 复制 `SKILL.md` 到 `~/.claude/skills/ensemble-deduction/`
2. 重启 Claude Code
3. 在多角色互动场景写作时，使用触发词："跑一下互动""角色推演""多agent"
